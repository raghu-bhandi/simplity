Application

Helps in Configuring the application

Example

<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<application xmlns="http://www.simplity.org/schema" applicationId="simplityTutorial" dbVendor="h2" dbDriverClassName="org.h2.Driver" connectionString="jdbc:h2:file:/db/classicmodels;user=sa;" sendTraceToClient="true" logSqls="true" >
</application>

Attributes

Attribute Required? Description
applicationId yes Unique name of this application within a corporate. This may be used as identity while trying to communicate with other applications within the corporate cluster
sendTraceToClient no Should we send server trace to client?
dbVendor no
The database vendor we are using
DBVendor: mysql, postgre, oracle, mssql, s2
cacheComponents no Do we cache components as they are loaded
connectionString no For connecting to data base, we either use connection string with driver class name, or use dataSource. Connection string overrides.
dbDriverClassName no The database driver to use
dataSourceName no Data source name to be used to look-up in JNDI for dataSource
loginServiceName no The service identifier used to perform login. If this is null, we simulate a dummy login where loginId is accepted as userId
logoutServiceName no The service identifier used to perform logout action. No action is taken if this is left as null
userIdIsNumber no Does this project use an integer as user id? (default is string)
sessionHelperClassName no Session helper is a class that implements org.simplity.http.WebSessionHelper interface. Simplity uses an instance of this class to manage session variables. Session management is also associated with login requirements. Simplity provides three default classes. HelperForNoLogin during development if session variables are not used, DefaultHelper that uses a simple session management, and PassiveHelper when this application uses Simplity only as a module, and session is not managed by Simplity. You may write your session manager based on the application design
cacheManagerClassName no Response for certain services can be cached. Also, during development, we may want to fake some services with a pre-determined response. Use a class that implements org.simplity.service.CacheManager interface
accessControllerClassName no Class that decides whether a userId be served a given service
exceptionListenerClassName no Way to wire exception to corporate utility
logSqls no Should the sqls that are executed be added to trace?? Required during development. Some corporates have security policy that requires you not log sqls
sessionParams no Session parameters
userIdNameInSession no Name of the session parameter that has the value for userId
schemaDetails no Some projects use multiple schema. In such a case, it is possible that a given service may use a schema other than the default. We have optimized the design for a frequently used default schema that is set for the user, and a few services use their own schema. Provide such schema with dataSource/connection
mediaStorageFolderPath no Simplity provides a rudimentary, folder-based system that can be used for Storing and retrieving attachments/media/files. If you want to use that, provide the folder that is available for the server instance
mediStorageAssistantClass no If you have an infrastructure for storing and retrieving attachments/media/files that you want to use, provide the class name that implements MediaStorageAssistant. A single instance of this class is re-used
